# 热键说明

本文档列出当前脚本已定义的热键及其行为前提。

## 程序启动与窗口操作

### `Win+Q`

行为：
- 向当前活动窗口发送 `Alt+F4`。
- 若窗口关闭后焦点回到桌面或任务栏，脚本会尝试激活当前桌面最上层的正常程序窗口。

注意：
- 如果关闭动作触发了程序自己的确认框、保存框或其他对话框，脚本会优先保留那个已经拿到焦点的窗口。

### `Win+Enter`

行为：
- 发送 `Win+3`，打开任务栏第 3 个程序。

### `Win+C`

行为：
- 发送 `Win+4`，打开任务栏第 4 个程序。

### `Win+I`

行为：
- 发送 `Win+5`，打开任务栏第 5 个程序。

### `Win+W`

行为：
- 发送 `Win+6`，打开任务栏第 6 个程序。

### `Win+O`

行为：
- 发送 `Win+7`，打开任务栏第 7 个程序。

### `Win+G`

行为：
- 发送 `Win+8`，打开任务栏第 8 个程序。

### `Win+M`

行为：
- 发送 `Win+9`，打开任务栏第 9 个程序。

### `Win+P`

行为：
- 直接启动：
  `"%USERPROFILE%\scoop\apps\postman\current\app\Postman.exe"`

注意：
- 如果本机未通过 Scoop 安装 Postman，或者安装路径不同，这个热键需要改路径。

## 输入法切换

### `RAlt`

行为：
- 在普通场景下，直接在微软拼音中文模式和英文模式之间切换。
- 在命中 `RAltOccupiedApps` 的程序里，不直接发 IME 消息，而是发送 `ImeToggleKey`。

典型用途：
- 日常中英文输入切换。
- 对 Excel/WPS 这类容易吞掉 `RAlt` 的程序做兼容。

### `Esc`

行为：
- 使用 `~Esc`，不会拦截原始 `Esc`。
- 当当前输入法处于微软拼音中文模式时，尽量切回英文模式。

典型用途：
- Vim、终端、编辑器命令模式切换时，降低误留中文输入状态的概率。

## 调试

### `Win+Shift+X`

行为：
- 显示当前活动窗口调试信息。

输出内容包括：
- 进程名
- 标题
- 类名
- HWND
- 当前 Layout ID
- 当前 IME 模式
- 当前配置中的英文和中文 Layout ID
- `ImeToggleKey`
- `RAltOccupiedApps` 是否命中

## 任务栏映射建议

如果你想让热键语义与注释保持一致，需要把应用固定到对应任务栏位置。当前注释表达的是：

- `Win+1`：企业微信
- `Win+2`：微信
- `Win+3`：终端
- `Win+4`：浏览器
- `Win+5`：IDEA
- `Win+6`：WebStorm
- `Win+7`：Zed
- `Win+8`：DBeaver
- `Win+9`：QQMusic

如果任务栏顺序不同，脚本行为也会跟着变化。
